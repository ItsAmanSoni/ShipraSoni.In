### Component API Design

**Consistent Props Pattern:**

```typescript
interface ComponentProps {
  // Visual variants
  variant?: 'solid' | 'outline' | 'ghost' | 'link';
  size?: 'sm' | 'md' | 'lg' | 'xl';
  colorScheme?: 'primary' | 'secondary' | 'success' | 'danger';

  // States
  isDisabled?: boolean;
  isLoading?: boolean;
  isReadOnly?: boolean;
  isRequired?: boolean;
  isInvalid?: boolean;

  // Layout
  fullWidth?: boolean;

  // Accessibility
  'aria-label'?: string;
  'aria-labelledby'?: string;
  'aria-describedby'?: string;

  // Common
  children?: React.ReactNode;
  className?: string;
  style?: React.CSSProperties;

  // Events
  onClick?: (event: React.MouseEvent) => void;
  onFocus?: (event: React.FocusEvent) => void;
  onBlur?: (event: React.FocusEvent) => void;
}
```

**Example: Button Component**

```typescript
// Button.tsx
import { forwardRef } from 'react';
import { styled } from '@emotion/styled';
import { tokens } from '../tokens';

interface ButtonProps extends ComponentProps {
  leftIcon?: React.ReactElement;
  rightIcon?: React.ReactElement;
  type?: 'button' | 'submit' | 'reset';
}

const StyledButton = styled.button<ButtonProps>`
  /* Base styles */
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: ${tokens.spacing.medium};
  padding: ${tokens.component.button.padding};
  border-radius: ${tokens.component.button.borderRadius};
  font-size: ${tokens.component.button.fontSize};
  font-weight: 600;
  transition: all 0.2s ease;
  cursor: pointer;
  border: none;

  /* Variant styles */
  ${({ variant, colorScheme }) => getVariantStyles(variant, colorScheme)}

  /* Size styles */
  ${({ size }) => getSizeStyles(size)}

  /* State styles */
  ${({ isDisabled }) => isDisabled && disabledStyles}
  ${({ isLoading }) => isLoading && loadingStyles}

  /* Focus styles for accessibility */
  &:focus-visible {
    outline: 2px solid ${tokens.color.focus};
    outline-offset: 2px;
  }
`;

export const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ children, leftIcon, rightIcon, isLoading, ...props }, ref) => {
    return (
      <StyledButton
        ref={ref}
        disabled={props.isDisabled || isLoading}
        aria-busy={isLoading}
        {...props}
      >
        {isLoading && <Spinner size="sm" />}
        {!isLoading && leftIcon}
        {children}
        {!isLoading && rightIcon}
      </StyledButton>
    );
  }
);

Button.displayName = 'Button';

Button.defaultProps = {
  variant: 'solid',
  size: 'md',
  colorScheme: 'primary',
  type: 'button',
};
```

const figmaAPI = require('figma-api');
const fs = require('fs');

const api = new figmaAPI({ personalAccessToken: process.env.FIGMA_TOKEN });

async function syncTokens() {
  // Fetch design tokens from Figma

## Challenges & Solutions

### Challenge 1: Migration from Legacy Systems

**Problem**: 15 applications using different UI frameworks needed gradual migration without breaking production.

**Solution - Incremental Migration Strategy**:

```typescript
// Wrapper component for gradual adoption
import { Button as NewButton } from '@client/design-system';
import { OldButton } from './legacy/Button';

export const Button = ({ useLegacy = false, ...props }) => {
  // Feature flag for gradual rollout
  const shouldUseLegacy = useLegacy || !window.FEATURE_FLAGS?.newDesignSystem;

  return shouldUseLegacy ? (
    <OldButton {...props} />
  ) : (
    <NewButton {...props} />
  );
};
```

**Results**:
- Zero downtime migrations
- Gradual feature flag rollout (10% â†’ 50% â†’ 100%)
- Rollback capability if issues detected
- All 15 apps migrated successfully over 4 months

## Lessons Learned

### Technical Learnings

**1. Start with Tokens, Not Components**
- Design tokens provided foundation for consistency
- Token-first approach enabled theming from day one
- Avoided rework when requirements changed

**2. Invest in Developer Experience**
- CLI tools accelerated adoption
- Good documentation === faster onboarding
- Storybook was invaluable for development and demos

**3. Accessibility Cannot Be an Afterthought**
- Built-in from day one vs. retroactive fixes
- Automated testing caught 90% of issues
- Saved months of remediation work

**4. Performance Optimization is Ongoing**
- Initial 850KB bundle was too large
- Continuous monitoring and optimization needed
- Bundle analysis should be part of CI/CD

**5. Type Safety Pays Off**
- TypeScript caught bugs before they reached production
- Improved IDE experience for all developers
- Self-documenting code through types

### Process Learnings

**1. Cross-functional Collaboration is Critical**
- Weekly syncs between design and engineering
- Shared Figma workspace improved communication
- Early involvement of QA prevented issues

**2. Incremental Rollout Reduced Risk**
- Feature flags enabled gradual adoption
- Canary deployments caught issues early
- Rollback plan gave confidence to move fast

**3. Documentation is Not Optional**
- Comprehensive docs led to faster adoption
- Examples reduced support burden
- Living documentation stayed up-to-date

**4. Version Management Matters**
- Semantic versioning with changelogs
- Deprecation warnings before breaking changes
- Migration guides for major versions

**5. Feedback Loops Accelerate Improvement**
- Monthly office hours with dev teams
- Slack channel for quick questions
- Quarterly surveys shaped roadmap
