## August 2020: The Nervous Beginning

The first day at Rubiq Technologies as a React.js Developer Intern felt overwhelming. Fresh from Geetanjali Institute of Technical Studies with theoretical knowledge but zero production experience, Aman stared at a codebase with 50+ components, Redux state management, and API integrations he'd only read about.

**The Reality Check:**
- Textbook knowledge: Solid
- Practical skills: Minimal
- Confidence: Low
- Imposter syndrome: Maximum

This is the unfiltered story of how that nervous intern became a full-stack developer shipping code to Fortune 500 companies across manufacturing, energy, and industrial sectors.

## Phase 1: Rubiq Technologies (Aug 2020 - March 2021)

### Month 1: Survival Mode

**First Week Tasks:**
- "Fix this button styling" (took 4 hours)
- "Add a loading spinner" (took 3 hours)
- "Update API endpoint" (broke production, took senior 10 minutes to fix)

**The Learning:** Pride comes before a fall. GitHub has a history tab for a reason.

**Key Moment:** After breaking production for the third time, the senior developer sat down and said, "Let me show you how to test locally first."

That conversation changed everything.

### Month 2-3: The Fundamentals

**Daily Routine:**
- 9 AM - 6 PM: Coding at Rubiq
- 7 PM - 11 PM: Online courses (Udemy, YouTube)
- Weekends: Building side projects

**What Clicked:**
- React hooks finally made sense
- Redux wasn't scary anymore
- API integration became routine
- Git branching strategy was clear

**Projects Built:**
- Todo app (cliché but essential)
- Weather app with API
- E-commerce clone
- Portfolio website

**Mistake:** Built everything from scratch without using libraries. Wasted time? No—understood fundamentals deeply.

### Month 4-7: Finding Confidence

**Breakthrough Project:** Company dashboard redesign

**Responsibilities:**
- 15 components
- Redux state management
- Axios API integration
- Responsive design

**Timeline:** 3 weeks (estimated 2 weeks, ran over)

**Result:**
- Client loved it
- Senior praised the code quality
- First pull request merged without changes

**The Moment:** Realized I could actually build things that matter.

### Key Lessons from Rubiq:

1. **Small Wins Matter**
   - Every fixed bug builds confidence
   - Every merged PR is progress
   - Comparison is the thief of joy

2. **Fundamentals > Frameworks**
   - JavaScript depth beats React breadth
   - Understanding "why" matters more than "how"
   - Basics never go out of style

3. **Senior Developers Are Teachers**
   - Ask questions (even "stupid" ones)
   - Code reviews are learning opportunities
   - Mentorship accelerates growth 10x

**Status at Exit:**
- Comfortable with React
- Understanding of full development cycle
- Ready for next challenge
- Confidence: Medium

## Phase 2: Cornice Info Solutions (Jun 2022 - Sep 2022)

### The Backend Shift

**New Role:** Full Stack Engineer Intern (Node.js focus)

**Initial Reaction:** Panic. Barely touched backend before.

**Week 1:** "What's middleware? What's MongoDB? What's REST?"

**Week 2:** "Oh, it's just JavaScript on the server. I can do this."

### Project: Food Delivery App Backend

**Tech Stack:**
- Node.js + Express
- MongoDB + Mongoose
- JWT authentication
- Payment gateway integration
- Real-time order tracking

**Timeline:** 4 months

**The Challenge:** Everything was new
- Database design
- Authentication flows
- Payment processing
- Error handling
- Security considerations

### Month 1: Database Design Hell

**First Attempt at Schema:**
```javascript
// Bad - everything in one collection
const orderSchema = new Schema({
  userId: String,
  userName: String,
  userEmail: String,
  userPhone: String,
  restaurantId: String,
  restaurantName: String,
  items: Array, // What's in here? Who knows!
  // ... 40 more fields
});
```

**Senior's Feedback:** "This is a nightmare to maintain. Let's talk about normalization."

**Second Attempt:** Relations, references, proper schema design

**Lesson:** Database design is harder than it looks. Plan before coding.

### Month 2-3: API Development

**Built 30+ API Endpoints:**
- User authentication (register, login, forgot password)
- Restaurant CRUD
- Menu management
- Order processing
- Payment integration
- Delivery tracking

**Mistakes Made:**
1. No input validation (fixed after hack attempt)
2. Exposed sensitive data in responses
3. No rate limiting (API got hammered)
4. Poor error messages (fixed after QA complaints)

**Fixes Applied:**
- Joi for validation
- Response sanitization
- Express rate limit
- Custom error handling middleware

### Month 4: Production Deployment

**The Night Before Launch:**
- Bug fixing until 2 AM
- Database migration scripts
- Environment variables setup
- SSL certificate configuration
- Load testing

**Launch Day:**
- 1000+ orders in first week
- 2 critical bugs found (fixed within hours)
- Database performance issues (added indexes)
- **Success:** App stayed up, users happy

### Key Lessons from Cornice:

1. **Backend Is Different**
   - Security is paramount
   - Data integrity matters
   - Performance at scale requires planning
   - Error handling saves lives

2. **Testing Isn't Optional**
   - Unit tests catch bugs early
   - Integration tests save production fires
   - Load testing prevents disasters

3. **Production Changes You**
   - Real users = real responsibility
   - Bugs at 2 AM teach lessons
   - Backups are non-negotiable

**Status at Exit:**
- Full-stack capable
- Backend confidence: High
- Production experience: Check
- Ready for bigger challenges

## Phase 3: Heylix IT Solutions (Sep 2022 - Apr 2023)

### The Architecture Evolution

**New Role:** Backend Engineer Intern (Microservices)

**The Project:** Agriculture platform with microservices architecture

**Tech Stack:**
- Sails.js (new framework)
- Microservices
- Docker
- Message queues
- Multiple databases

**First Thought:** "What have I gotten myself into?"

### Learning Microservices the Hard Way

**Week 1:** Reading microservices patterns

**Week 2:** Building first service (User Service)

**Week 3:** Realizing microservices are complex
- Service discovery
- Inter-service communication
- Data consistency
- Distributed logging
- Monitoring

**Week 4:** "This is why Netflix engineers are paid well."

### Project Breakdown:

**Services Built:**
1. User Service (Authentication, profiles)
2. Product Service (Inventory management)
3. Order Service (Purchase processing)
4. Notification Service (Email, SMS)
5. Analytics Service (Reporting)

**Challenges Faced:**

**Challenge 1: Service Communication**
- REST? gRPC? Message queues?
- Synchronous? Asynchronous?
- What if service is down?

**Solution:** Event-driven architecture with RabbitMQ

**Challenge 2: Database Per Service**
- How to get user data in Order Service?
- How to maintain data consistency?
- How to handle transactions?

**Solution:** Event sourcing + CQRS patterns (complex but powerful)

**Challenge 3: Debugging**
- Logs spread across 5 services
- How to trace a request?
- Which service failed?

**Solution:** Distributed tracing with correlation IDs

### The Dashboard Project

**Parallel Project:** Car company dashboard (React + Atlassian)

**Back to Frontend:** Felt like coming home

**Built:**
- Real-time analytics dashboard
- 20+ data visualizations
- Responsive design
- Role-based access control

**Technologies:**
- React with hooks
- Atlassian UI Kit
- Chart.js
- WebSockets for real-time updates

**Timeline:** 6 weeks

**Result:** Client impressed, extended contract

### Key Lessons from Heylix:

1. **Architecture Matters**
   - Monolith vs Microservices trade-offs
   - No silver bullets
   - Complexity has a cost

2. **Full-Stack Means Both**
   - Don't abandon frontend for backend
   - T-shaped skills beat I-shaped skills
   - Versatility is valuable

3. **Production at Scale**
   - 100 users ≠ 10,000 users
   - Monitoring is critical
   - Plan for failure

**Status at Exit:**
- Microservices experience: Check
- Architecture understanding: Good
- Problem-solving: Strong
- Ready for enterprise
